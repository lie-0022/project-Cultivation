# 05. 교배 시스템 (Breeding)

## 1. Overview

크리처 2마리를 조합해 새로운 크리처를 만드는 시스템. 게임의 핵심 재미 포인트.

## 2. Player Fantasy

"이 둘을 섞으면 뭐가 나올까?" — 새로운 종 발견의 실험적 재미.

## 3. Detailed Rules

### 3.1 흐름

1. 사육장 UI 또는 교배 연구소 시설물에서 진입
2. 크리처 A 선택 (Busy 아닌 것만)
3. 크리처 B 선택 (Busy 아닌 것만, A와 동일 인스턴스 불가)
4. "교배 시작" 버튼 클릭
5. 두 크리처 모두 `isBusy = true` 설정
6. `BreedingTask` 생성, 활성 태스크 리스트에 추가
7. 매 프레임 `elapsedTime += Time.deltaTime`
8. `elapsedTime >= requiredTime` → 결과 처리:
   - 결과 크리처 생성 (4.1 참조)
   - 사육장 빈 슬롯에 추가
   - 부모 두 마리 `isBusy = false` (잔존)
   - 빈 슬롯 없으면? → 5절 Edge Cases 참조

### 3.2 동시 교배

- 동시 진행 가능한 교배 슬롯 수 제한 없음 (무제한)
- 사육장의 모든 가능한 쌍을 동시에 교배 시작 가능

### 3.3 교배 취소

- 활성 교배 태스크는 언제든 취소 가능
- 취소 시:
  - 두 부모 크리처 `isBusy = false` (즉시 사용 가능)
  - 골드 지출이 있다면 전액 환불 (MVP는 교배 비용 없음, 미래 대비)
  - 결과물 없음

### 3.4 교배 완료 후 부모

- **잔존**: 교배 완료 시 두 부모 모두 사육장에 남음, `isBusy = false` 해제
- 같은 부모로 무제한 반복 교배 가능

### 3.5 비목표 (MVP 제외)

- 형질 유전 (색/크기 변이)
- 근친 교배 패널티
- 교배 비용 (MVP는 무료)
- 교배 시간 단축 아이템

## 4. Formulas

### 4.1 결과 결정 로직

```
1. BreedingRecipe 리스트에서 검색:
   - parentA.creatureId, parentB.creatureId가 레시피와 매칭되는지 확인
   - 매칭 시 순서 무관 (A+B == B+A)
2. 매칭되는 레시피 발견 → resultCreatureId의 크리처 생성
3. 매칭 없음 → 부모 중 하나 무작위 복제 (50% 확률):
   - roll = UnityEngine.Random.value
   - if (roll < 0.5f) clone parentA.creatureId
   - else clone parentB.creatureId
```

### 4.2 우선순위 규칙 (#8 결정)

- **레시피 우선**: 같은 종 두 마리(예: 토마토닭 + 토마토닭)도 레시피에 명시되어 있으면 그 결과(케첩닭) 적용
- 레시피가 없을 때만 자기복제(부모 복제) 분기로 진입

### 4.3 교배 시간

- 각 `BreedingRecipe`에 `breedingTime` 필드로 정의
- 자기복제(레시피 없음) 케이스의 시간: **두 부모 종이 등장하는 가장 짧은 레시피의 시간**, 없으면 기본값 30초 (MVP는 단순화)

> **결정 필요 시점**: 자기복제 시간 처리 방식이 추가로 모호하면 추후 재논의. MVP는 일단 "기본값 30초" 사용.

## 5. Edge Cases

- **사육장 가득 참 + 교배 완료**: 결과 크리처 임시 보관 슬롯에 대기 표시 (UI에서 알림). 사용자가 사육장 비우거나 확장하면 자동 추가. 또는 단순화: "사육장이 가득 차면 결과를 받을 수 없다" 알림 후 결과 폐기 (MVP는 후자 권장)
- **같은 인스턴스 두 번 선택**: 차단 (instanceId 동일성 체크)
- **교배 중 크리처를 다른 작업에 사용 시도**: Busy 플래그로 차단
- **교배 중 게임 종료**: 알림 표시 후 종료, 다음 게임 시 모든 진행 사라짐 (MVP는 저장 없음)
- **레시피의 parentA_Id == parentB_Id (같은 종 자기 교배 레시피)**: 정상 동작, 사육장에 같은 종 2마리 있어야 시작 가능

## 6. Dependencies

- `BarnManager` — 부모 선택, 결과 추가
- `CreatureInstance` — 부모 인스턴스, isBusy 토글
- `CreatureData` — 종 식별
- `BreedingRecipe` — 조합 정의
- `EconomyManager` — (미래) 교배 비용/환불

## 7. Tuning Knobs

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `breedingTime` (recipe별) | 60~90초 | 각 레시피의 교배 시간 |
| 자기복제 기본 시간 | 30초 | 레시피 매칭 없을 때 |
| 자기복제 확률 | 50% / 50% | 부모 A vs 부모 B |

## 8. Acceptance Criteria

- [ ] 사육장에서 크리처 2마리 선택 후 교배 시작 가능
- [ ] 같은 인스턴스 두 번 선택 차단
- [ ] 교배 시작 시 두 부모 isBusy = true
- [ ] 교배 진행 시간이 실시간으로 흐름
- [ ] 레시피 매칭 시 정확한 결과 크리처 생성
- [ ] 레시피 매칭 없을 때 부모 중 하나 복제 (50% 확률)
- [ ] 같은 종 2마리도 레시피에 있으면 레시피 결과 우선
- [ ] 교배 완료 후 두 부모 isBusy = false (잔존)
- [ ] 동시에 여러 교배 진행 가능
- [ ] 교배 취소 시 부모 즉시 isBusy = false
- [ ] 사육장 가득 참 시 결과 처리 정책대로 동작 (MVP: 폐기 + 알림)

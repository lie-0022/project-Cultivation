# 04. 사육장 시스템 (Barn)

## 1. Overview

크리처를 보관하고 관리하는 시스템. 교배의 출발점이자 판매의 출발점.

## 2. Player Fantasy

내가 만든 크리처들이 모여 있는 컬렉션 공간.

## 3. Detailed Rules

### 3.1 슬롯 구조

- 각 사육장 슬롯은 0개 또는 1개의 `CreatureInstance` 보유
- 시작 슬롯 수: 2
- 골드로 슬롯 추가 가능 (상한 없음)

### 3.2 흐름

1. 사육장 시설물 접근 + E키 → 사육장 UI 열림
2. 슬롯 목록 표시 (각 슬롯의 크리처 정보, Busy 상태 표시)
3. 가능한 액션:
   - **크리처화**: 작물을 새 크리처로 변환 (03 GDD 참조)
   - **교배**: 크리처 2마리 선택 → 교배 시작 (05 GDD 참조)
   - **판매**: 크리처 1마리 선택 → 판매 (06 GDD 참조)
   - **사육장 확장**: 골드로 슬롯 추가
4. Busy 상태 크리처는 교배/판매에서 선택 불가

### 3.3 확장

- 골드를 사용해 새로운 슬롯 추가 구매
- 시작 칸 수: 2
- 상한 없음 (MVP)

## 4. Formulas

### 4.1 확장 비용

```
nextCost = Mathf.RoundToInt(barnBaseCost * Mathf.Pow(currentSlotCount, costMultiplier))
```

- `currentSlotCount`: 현재 보유한 슬롯 수 (직전 칸 수)
- 예시: 2칸 → 다음 비용 = `200 * (2^1.5)` ≈ 566 골드

기본값:
- `barnBaseCost = 200`
- `costMultiplier = 1.5`

## 5. Edge Cases

- **모든 슬롯 가득 + 크리처화 시도**: 차단
- **확장 시 골드 부족**: 구매 차단
- **Busy 크리처를 교배/판매에 선택 시도**: 차단, "이 크리처는 교배 중" 메시지

## 6. Dependencies

- `EconomyManager` — 확장 비용 차감
- `CreatureInstance` — 슬롯에 보관되는 런타임 객체
- `CreatureData` — 크리처 정보 표시용
- `ExpansionConfig` — 확장 비용 공식
- 03 GDD (크리처화), 05 GDD (교배), 06 GDD (판매) — 사육장에서 진입

## 7. Tuning Knobs

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| 시작 슬롯 수 | 2 | 게임 시작 시 사육장 슬롯 개수 |
| `barnBaseCost` | 200 | 확장 비용 베이스 |
| `costMultiplier` | 1.5 | 확장 비용 지수 |

## 8. Acceptance Criteria

- [ ] 사육장 UI에서 모든 슬롯과 보관 크리처가 정상 표시된다
- [ ] 빈 슬롯과 채워진 슬롯이 시각적으로 구분된다
- [ ] Busy 상태 크리처는 시각적으로 구분되며 교배/판매에서 선택 불가
- [ ] 골드로 새 슬롯 추가 가능, 비용이 공식대로 계산된다
- [ ] 골드 부족 시 확장 차단된다
- [ ] 사육장에서 크리처화/교배/판매 액션이 모두 진입 가능하다
